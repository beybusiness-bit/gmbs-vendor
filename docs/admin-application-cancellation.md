# 어드민 참조 문서: 신청 취소 처리 방식 변경

## 변경 내용 요약

기존에는 벤더 포탈에서 사용자가 **신청 취소** 버튼을 누르면 Firestore 문서가 **삭제(deleteDoc)** 되었습니다.

변경 후에는 문서를 삭제하지 않고 **`status: '취소됨'`** 으로 업데이트합니다. 취소 일시는 `cancelled_at` 필드에 기록됩니다.

---

## 적용 대상 컬렉션

| 컬렉션 | 신청 유형 |
|---|---|
| `brand_applications` | 새 브랜드 등록 신청 |
| `brand_join_requests` | 기존 브랜드 합류 신청 |

---

## 상태값 전체 목록

### brand_applications

| status | 의미 |
|---|---|
| `제출됨` | 심사 대기 중 |
| `승인` | 어드민 승인 완료 |
| `거절` | 어드민 거절 |
| `취소됨` | **신청자가 직접 취소** (신규 추가) |

### brand_join_requests

| status | 의미 |
|---|---|
| `제출됨` | 검토 대기 중 |
| `승인` | 어드민 또는 벤더 주관리자가 승인 |
| `거절` | 어드민 또는 벤더 주관리자가 거절 |
| `취소됨` | **신청자가 직접 취소** (신규 추가) |

---

## 어드민 대응 사항

### 1. 취소된 신청 필터링

어드민에서 신청 목록을 표시할 때 `status: '취소됨'` 항목을 별도 탭 또는 필터로 구분하는 것을 권장합니다.

- **처리 필요 목록**: `status == '제출됨'` 만 표시
- **전체 내역**: `취소됨`, `거절`, `승인` 포함

### 2. 기존 삭제 처리와의 차이

| 구분 | 기존 (삭제) | 변경 후 (취소됨) |
|---|---|---|
| 문서 존재 여부 | 삭제됨 | 유지됨 |
| 어드민 열람 | 불가 | 가능 |
| 취소 주체 | - | `cancelled_at` 필드로 시점 확인 가능 |
| 어드민 삭제 | 별도 삭제 가능 | 기존과 동일하게 별도 삭제 가능 |

어드민이 직접 문서를 삭제하는 기능은 별개이며 이번 변경과 무관합니다.

### 3. 알람 처리

`brand_join_requests`의 경우, 취소됨 처리 시 어드민 알람이 자동 해소되어야 합니다.  
어드민 알람 로직에서 `status: '취소됨'` 인 건을 "처리 불필요"로 분류해 주세요.

### 4. 중복 신청 방지

신청자가 `취소됨` 상태의 신청을 재신청하는 것은 허용됩니다.  
`취소됨` 건은 현재 진행 중인 신청이 아니므로 `제출됨` 상태의 건이 없다면 새 신청을 받을 수 있습니다.

## Firestore 규칙 변경 사항

신청자 본인이 `취소됨`으로 상태를 변경할 수 있도록 update 규칙이 추가되었습니다.

### brand_applications
```js
// 신청자 본인이 '제출됨' → '취소됨'으로만 변경 가능
// 변경 가능 필드: status, cancelled_at
allow update: if request.auth != null
  && resource.data.applicant_uid == request.auth.uid
  && resource.data.status == '제출됨'
  && request.resource.data.status == '취소됨'
  && request.resource.data.diff(resource.data).affectedKeys()
       .hasOnly(['status', 'cancelled_at']);
```

### brand_join_requests
기존 update 규칙(브랜드 담당자의 승인/거절)에 추가하여,  
신청자 본인의 취소 처리 규칙이 OR 조건으로 추가되었습니다.
